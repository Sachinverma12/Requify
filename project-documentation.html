1. Architecture Overview

Single-page-inspired architecture using static HTML files served by Express, with a JSON-file-based REST API.

🌐 Browser
HTML/CSS/JS served statically
→
📦 Express Server
Static file host + REST API on :5000
→
📁 JSON Files
users.json + properties.json

Data Flow

  1. Auth: Login/Signup → POST /api/auth/* → JWT issued → stored in localStorage
  2. Protected API: Every subsequent request includes Authorization: Bearer <token> header
  3. Role gating: Middleware checks req.user.role — user/owner/admin get different access levels
  4. Search: GET /api/properties?search=...&type=...&bedrooms=... → server filters in-memory
  5. Owner CRUD: POST/PUT/DELETE /api/owner/properties — scoped to ownerId
  6. Admin: Full access to all users and properties via /api/admin/*

Production Deployment Checklist

  1. Generate a strong JWT secret: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  2. Set environment variables: Copy .env.example → .env and fill in production values:
    • JWT_SECRET = the generated secret
    • CORS_ORIGIN = your frontend domain (e.g., https://requify.com)
    • NODE_ENV = production
    • PORT = your server port
  3. Upgrade storage: Replace JSON-file I/O with a database (MongoDB/PostgreSQL) for concurrent access safety
  4. Enable HTTPS: Use a reverse proxy (nginx, Caddy) or a service like Cloudflare
  5. Run with process manager: Use PM2 or systemd to keep the server alive
  6. Configure config.js: Update BASE_URL to your production API domain
